# FAQ

## Нужно ли хранить token в `config/finam.php`?

Нет.  
Это сознательное правило пакета. Токен всегда передаётся явно в runtime.

## Чем отличается `secret` от `token`?

- `secret` нужен, чтобы получить session token
- `token` нужен, чтобы уже выполнять запросы к API

Обычно flow такой:

1. `Finam::issueToken($secret)`
2. `Finam::connect($token)`

Если не нужен ручной контроль над промежуточным токеном, можно короче:

1. `Finam::connectSecret($secret)`

## Когда использовать `Finam::connect($token)`?

Почти всегда.  
Это основной и самый удобный способ работы с пакетом.

Если у тебя на входе именно `secret`, то ещё удобнее использовать `Finam::connectSecret($secret)`.

## Когда использовать `Finam::client($token)`?

Когда нужен прямой REST-доступ и ты хочешь сам работать с `ApiResponse`.

## Почему часть методов принимает `accountId`, а часть нет?

Потому что не все endpoint-ы Finam требуют счёт.

Примеры:

- `getExchanges()` не требует `accountId`
- `getOrders()` требует контекст счёта

## Что будет, если не передать `accountId`?

Если в сессии ровно один счёт, пакет подставит его сам.  
Если счетов несколько, будет `AccountResolutionException`.

## Почему числа часто приходят как строки?

Потому что для торговых API это безопаснее.  
Цены, количества и другие точные значения лучше не терять через лишние float-преобразования.

## Почему я получил `403`, хотя код вроде правильный?

Самые частые причины:

- у токена нет нужных прав
- у брокерского счёта нет доступа к операции
- метод доступен, но не для этого типа аккаунта

Это не всегда ошибка SDK.

## Почему отчёт не создаётся с моим `reportForm`?

Потому что допустимые значения зависят от Finam и могут меняться.  
SDK поддерживает сам report flow, но конкретное значение `reportForm` нужно сверять с актуальной документацией и возможностями твоего аккаунта.

## Можно ли использовать пакет без Laravel?

Да.  
Проще всего через `FinamClient::make($token)`.

## Есть ли в пакете streaming?

Нет.  
Сейчас основной акцент на typed REST API.

## Есть ли в пакете бизнес-логика торговли?

Нет.  
Пакет умеет обращаться к API, но не принимает торговые решения за тебя.
